    Introduction

    Welcome to Pacman.Trade Developer Documentation. It is the only official documentation of Pacman.Trade API.

    This documentation provides an introduction to the use of related APIs.

    RESTful API includes interfaces such as trading pairs info and orders.

    The services provided by Pacman.Trade API will be continuously updated here, so stay tuned for updates.

    Quick Start

    REST API

    REST, the abbreviation of Representational State Transfer, is a popular Internet transmission architecture. It has the characteristics of clear structure, standards compliance, easy understanding and expansion, and is being adopted by more and more websites with the benefits as follows:

    • In RESTful architecture, each URL represents a resource;
    • Some representation layer of this resource is transferred between the client and the server;
    • The client operates the server-side resources through four HTTP instructions to realize "presentation layer state transformation".

    Developers are advised to use REST API for one-time operations such as trading pairs info or orders.

    Access URL

    REST API

    https://api.pacman.trade/

    Rate Limit

    Same IP

    A maximum of 300 requests is allowed every 3 seconds for the same IP.

    Same User

    For the same user, a maximum of 120 requests is allowed every 3 seconds.

    API Access Instructions

    Request Format

    All API requests are restful, and there are only two methods at present: GET and POST. - GET request: All parameters are in path parameters - POST request: Parameters can be set in the path, and they can be sent in JSON format to the request body. If there are no parameters, {} needs to be sent

    A licit request consists of the following parts: method request address: Access server address api.pacman.trade，e.g. https://api.pacman.trade/market/symbol-thumb & Required and optional parameters

    Note: 'Content-Type':'application/json' needs to be set.

    Return Format

    All interface returns are in JSON format.

    Error Message

    HTTP Status Code

    Common error codes for HTTP are as follows: - 400 Bad Request – Invalid request format

    • 404 service not found

    • 429 too many visits

    • 500 internal server error

    Basic Information

    Currency & Spot Info

    Get Currency & Spot Info

    HTTP Request:

    • POST /market/symbol-thumb

    Request Params

    param type required comment
    - - - -

    Response Data

    code type comment
    symbol String Symbol name
    time Long Listing time, Unix timestamp format in milliseconds, e.g. 1597026383085
    open Float Service field (not in use)
    high Float 24h high
    low Float 24h low
    close Float Current price
    chg Float Daily change in %
    change Float Service field (not in use)
    volume Float 24h volume
    turnover Float Turnover
    lastDayClose Float Service field (not in use)
    usdRate Float Currency price in USD
    baseUsdRate Float Base USD Price : 1
    zone Integer Service field (not in use)

    Request Parameters

    param type required comment

    Response Data

    code type comment
    "ask" String Defining the info about orders on sell
    - symbol String Currency name
    - minAmount Float Minimal amount of orders with one price
    - maxAmount Float Maximal amount of orders with one price
    - highestPrice Float Highest price of order
    - lowestPrice Float Lowest price of order
    - direction String Showing the direction of order ("SELL"/"BUY")

    - - price Float Price of order
    - - amount Float Amount of currency in all orders with corresponding price
    "bid" String Defining the info about orders on buy
